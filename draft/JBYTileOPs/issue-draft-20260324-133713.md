# [PERF][Pooling] Optimize MaxPool2d kernel to match or exceed PyTorch performance

## Description

### Symptom / Motivation
MaxPool2d kernel correctness is verified (10 tests pass), but performance is significantly slower than PyTorch baseline:
- TileOps: ~0.08ms for (batch=1, channels=3, H=224, W=224, kernel=2, stride=2)
- PyTorch: ~0.016ms
- Speedup: 0.19x (TileOps is 5x slower)

The current kernel uses a simple 1-thread-1-output approach without shared memory or thread cooperation, resulting in poor memory access patterns and redundant loads.

### Root Cause Analysis
- `tileops/kernels/pooling/max_pool2d.py:42-75` - Current kernel structure:
  - Uses `T.Kernel(batch * channels * out_h * out_w, threads=threads)` with linear pid
  - Each thread independently loads up to 4 kernel elements from global memory
  - No shared memory buffering between threads
  - No warp-level or block-level reduction

- Reference architecture: `tileops/kernels/max_pool2d/max_pool2d.py` (1D pooling) uses:
  - Blocked thread cooperation with `block_b` parameter
  - `T.alloc_shared()` for kernel window data
  - `T.alloc_fragment()` for local accumulation
  - `T.copy()` for shared→local transfers

### Related Files
- `tileops/kernels/pooling/max_pool2d.py` — MaxPool2d kernel implementation (needs optimization)
- `tileops/kernels/max_pool2d/max_pool2d.py` — 1D pooling reference architecture
- `tileops/kernels/deepseek_nsa/mean_pooling_fwd.py` — mean pooling with shared memory pattern
- `tileops/ops/pooling/max_pool2d.py` — Op wrapper
- `benchmarks/ops/bench_max_pool2d.py` — benchmark

## Goal
Achieve ≥1.0x speedup (match or exceed PyTorch) for MaxPool2d kernel on representative workloads (224x224, 112x112, 56x56) with FP16/BF16 dtypes.

## Plan
**Plan type: proposal**

1. **Analyze reference implementation**: Study `tileops/kernels/max_pool2d/max_pool2d.py` blocked architecture and adapt for 2D pooling
   - Understand block_bc blocking strategy
   - Apply shared memory loading pattern

2. **Rewrite kernel with shared memory** in `tileops/kernels/pooling/max_pool2d.py`
   - Use `T.Kernel(T.ceildiv(batch * channels, block_bc), out_h, out_w, threads=threads)`
   - Pre-load kernel window into `T.alloc_shared()` buffer
   - Use `T.copy()` for shared→fragment transfer
   - Implement warp-level max reduction

3. **Extend autotune configs** to find optimal (block_bc, threads) combinations
   - block_bc: [1, 2, 4, 8, 16]
   - threads: [32, 64, 128, 256, 512]

4. **Validate and benchmark**
   - Ensure all 10 existing tests pass
   - Run benchmark vs PyTorch on 224x224, 112x112, 56x56 workloads

## Constraints

- Must not break existing public API
- Must support FP16 and BF16 dtypes
- Must follow TileOPs 2-layer architecture (Kernel L1 → Op L2)
