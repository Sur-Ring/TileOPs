# [FEAT][Pooling] Implement Average Pooling (AvgPool) Forward Operator

## Description

### Symptom / Motivation
The conv & pooling operator family tracking issue #402 requires implementing Average Pooling operators. AvgPool1d/2d/3d forward operators are needed to complement the pooling operator family, similar to PyTorch's `torch.nn.functional.avg_pool1d/2d/3d`.

### Root Cause Analysis
N/A - new feature implementation

### Related Files

**Reference APIs (PyTorch)**:
- `torch.nn.functional.avg_pool1d` ‚Äî input `(N, C, L)`, output `(N, C, L_out)`
- `torch.nn.functional.avg_pool2d` ‚Äî input `(N, C, H, W)`, output `(N, C, H_out, W_out)`
- `torch.nn.functional.avg_pool3d` ‚Äî input `(N, C, D, H, W)`, output `(N, C, D_out, H_out, W_out)`

**Reference implementation pattern**:
- `tileops/kernels/deepseek_nsa/mean_pooling_fwd.py` ‚Äî TileLang kernel + torch.library.custom_op pattern
- `tileops/kernels/kernel.py` ‚Äî Kernel base class

**Files to create**:
- `tileops/kernels/pooling/avg_pool1d.py` ‚Äî **new**: AvgPool1d Kernel
- `tileops/kernels/pooling/avg_pool2d.py` ‚Äî **new**: AvgPool2d Kernel
- `tileops/kernels/pooling/avg_pool3d.py` ‚Äî **new**: AvgPool3d Kernel
- `tileops/ops/pooling/avg_pool1d.py` ‚Äî **new**: AvgPool1d Op
- `tileops/ops/pooling/avg_pool2d.py` ‚Äî **new**: AvgPool2d Op

- `benchmarks/ops/bench_avg_pool2d.py` ‚Äî **new**: AvgPool2d benchmarks
- `benchmarks/ops/bench_avg_pool3d.py` ‚Äî **new**: AvgPool3d benchmarks

## Goal

Implement AvgPool1d, AvgPool2d, and AvgPool3d forward operators with standard pooling semantics (kernel_size, stride, padding, dilation), TileLang-based kernels, and PyTorch reference validation.

## Plan

**Plan type: proposal**

### Phase 1: AvgPool1d

1. **Kernel**: Create `tileops/kernels/pooling/avg_pool1d.py` implementing `AvgPooling1dKernel`
   - Input: `(N, C, L)`, Output: `(N, C, L_out)`
   - Formula: `L_out = (L + 2*padding - dilation*(kernel_size-1) - 1) // stride + 1`
   - Use `tilelang.jit` decorator and `torch.library.custom_op`
   - Replace max reduction with sum + divide (mean) reduction

2. **Op**: Create `tileops/ops/pooling/avg_pool1d.py` implementing `AvgPooling1dOp`

3. **Tests**: Create `tests/ops/test_avg_pool1d.py`
   - FP16 (rtol=1e-3, atol=1e-3)
   - BF16 (rtol=1.6e-2, atol=1.6e-2)
   - Reference: `torch.nn.functional.avg_pool1d`

4. **Benchmarks**: Create `benchmarks/ops/bench_avg_pool1d.py`
   - Report latency, TFLOPS, DRAM bandwidth
   - Compare against `torch` baseline

### Phase 2: AvgPool2d
